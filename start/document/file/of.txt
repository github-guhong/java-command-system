命令介绍
    of => openFile ，为了把文件分的清楚，往往会建立很多文件来分组，久而久之，目录层级越来深
    用起来的时候也特别不爽，所以可以通过[of 文件名]的方式快速打开你想要的文件
    原理很简单，指定你经常要用的目录，程序会扫描该目录所有文件/文件夹然后建立索引，如果有些文件不需要也可以忽略

命令用法
    of [reload 重新加载的目录...] [-i 忽略的值...] [-s 快捷目录...] [打开的文件/文件夹...] [-f/-d]
    快捷目录：
        就是指添加到系统中的目录，系统会为这些目录创建索引。
    如：
        1、of -s E:/笔记 -i .jpg,.png,node_modules
            # 表示将`E:/笔记`目录设为快捷目录，且忽略.jpg、.png后缀的文件和文件名为node_modules的文件或目录
        2、of 博客
            # 从添加的快捷目录中找名为[博客]的文件或目录
        3、of reload
            # 如果添加了新的文件，需要reload重新构建索引

命令参数解释
    reload:
        重新扫描指定目录，重新构建索引。多个以逗号隔开（英文逗号）。如果新增了文件可以使用该命令重新构建索引。
        可以指定扫描的目录，如果不指定就扫描所有现存的快捷目录。
        如果指定的目录没有设定为快捷目录，则会自动添加该目录为快捷目录。
    -i:
        设置忽略的文件/文件夹。多个以逗号隔开（英文逗号）
        支持三种方式：
            1、指定某一个文件/目录。如：E:\笔记\编程\前端\vue。表示忽略“E:\笔记\编程\前端\vue”这个文件夹，同样也可以指定文件
            2、指定某个名字的文件/目录。如：node_modules。表示忽略所有名字叫做node_modules的文件和目录
            3、指定某个文件后缀。如：.git。表示忽略所有以“.git”结尾的文件。
    -s：
        保存快捷目录。多个以逗号隔开（英文逗号）
        如果保存的目录是一个新目录则自动扫描这个路径并创建索引
    -f:
        表示只查询文件索引，不查询目录，不能和-d参数同时出现
    -d：
        表示只查询目录索引，不查询文件，不能和-f参数不能同时出现
    -il:
        查询当前设为忽略的文件/文件夹列表
    -sl:
        查询当前保存好的快捷目录列表
    -fil:
        查询当前所有索引列表
    -e:
        equal，采用完全匹配。

命令值解释
    指定一个文件名，如果查到多个相似文件名则打印查到的文件索引并提供选择，如果选择中没有你想要的，
    可以继续输入要打开的文件

注意
    1、不要加太多文件进去，扫描的时候会慢。如果想要大量产生文件索引，建议使用Everything。
    2、一定要过滤哪些依赖库。比如npm下载的“node_modules”文件，这里面文件非常多。
    3、可以多个参数一起使用，最终执行的顺序是：添加忽略路径——>添加快捷目录——>重新加载目录
    4、要打开的文件必须是第一个值。如：of 笔记 -f ，查询的就是【笔记】
